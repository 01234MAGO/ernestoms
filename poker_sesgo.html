SESGO DE SUPERVIVENCIA EN EL PÓKER

SOBRE ESTE PROYECTO

El objetivo de este proyecto es aplicar el enfoque del sesgo de supervivencia a mi estrategia y habilidades en el póker. La idea consiste en recolectar datos que sirvan como materia prima para realizar diversos análisis, como la identificación de patrones de juego, y así tener argumentos para tomar decisiones informadas durante una partida.
Un objetivo más ambicioso de este proyecto es crear un modelo predictivo para conocer las probabilidades aproximadas de éxito antes de comenzar cada mano.

RESEÑA HISTORICA

Durante la Segunda Guerra Mundial, los Aliados cartografiaron los agujeros de bala en los aviones que fueron alcanzados por el fuego nazi con el objetivo de fortalecerlos. Su enfoque consistía en reforzar las áreas más afectadas por la artillería enemiga para aumentar la resistencia de los aviones frente a futuros ataques.
Sin embargo, Abraham Wald llegó a una conclusión diferente que desafiaba el pensamiento convencional. Según su razonamiento, las zonas de mayor impacto representaban el daño de los aviones que lograron regresar a casa pudieron resistir. En lugar de reforzar las áreas dañadas, Wald argumentó que se debía prestar atención a las zonas que no mostraban impactos de bala. Estas áreas eran críticas, ya que los aviones que recibían disparos en esos puntos no lograban sobrevivir. Este fenómeno se conoce como sesgo de supervivencia, que ocurre cuando nos enfocamos en lo que ha sobrevivido en lugar de considerar lo que no lo ha hecho.
El enfoque toma la idea de que los datos disponibles solo representan una parte de la imagen completa y que se deben tener en cuenta las muestras faltantes para obtener conclusiones más precisas.

COMO MEJORAR EL MODELO?

Para mejorar el rendimiento del algoritmo, se pueden seguir dos enfoques complementarios. El primero es aumentar la cantidad de registros disponibles. El segundo es incorporar una variable clave, como las cartas del jugador. Dado que existen 1,326 combinaciones posibles de cartas y la cantidad actual de registros es apenas el doble, el algoritmo solo ha tenido la oportunidad de ver un número muy limitado de combinaciones. Por lo tanto, no podrá aprender adecuadamente de esta variable hasta que se disponga de una mayor cantidad de datos.

REPOSITORIO DEL PROYECTO

HABILIDADES INVOLUCRADAS

INGENIERIA DE DATOS

Para extraer datos relevantes, desarrollé un software que procesa archivos de texto. Utilizando una librería especializada, el software analiza fragmentos de texto para identificar y extraer la información necesaria, que luego se descarga en el formato requerido para su carga en una base de datos.
Los datos provienen de la plataforma de juego ‘POKERSTARS’, que proporciona información sobre cada mano en la que participa el usuario. La base de datos resultante está compuesta por 62 variables que reflejan diversas características del usuario durante cada mano.

ANALISIS DE DATOS

En esta etapa, comencé por clasificar las variables en numéricas, continuas, clasificatorias y de tiempo. Luego, analicé las relaciones entre estas variables para entender cómo se interrelacionan. Este análisis incluyó la identificación de patrones y correlaciones que podrían influir en el rendimiento del modelo. Finalmente, evalué qué variables serían más relevantes para entrenar un modelo predictivo, considerando su capacidad para proporcionar información valiosa y mejorar la precisión de las predicciones futuras.

MACHINE LEARNING

Opté por desarrollar un modelo de aprendizaje supervisado, comenzando por identificar la variable objetivo, que en este caso es categórica. Luego, procedí a codificar y estandarizar los datos, y a dividirlos en conjuntos de entrenamiento y prueba, una etapa esencial para que el algoritmo pueda comprender y generalizar correctamente a partir de los datos.
Con los datos ya preparados, comencé el proceso de entrenamiento de varios modelos de clasificación. Evalué cada uno mediante el cálculo de métricas clave y utilicé técnicas como la búsqueda de hiperparámetros óptimos para ajustar y mejorar el rendimiento del modelo. El objetivo fue desarrollar un modelo lo más fiable y preciso posible, capaz de hacer predicciones efectivas basadas en los datos disponibles.

EVALUACION DEL MODELO

El modelo seleccionado fue Random Forest. Aunque presenta un caso de sobreajuste, demostró ser el más eficaz para realizar predicciones en este contexto.
Es importante destacar que el modelo no es particularmente útil para predecir si se ganará una mano; sin embargo, su principal utilidad radica en identificar situaciones en las que las condiciones no son favorables para participar en la mano. En este aspecto, el algoritmo ha mostrado un buen desempeño.
